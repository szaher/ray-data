## Why serialization matters

When you call `ds.map(my_function)`, Ray needs to send `my_function` to every worker. Workers run as separate Python processes, potentially on different machines. The only way to send a Python object between processes is to convert it to bytes (serialize), send those bytes, and reconstruct the object on the other side (deserialize).

<MermaidDiagram chart={`sequenceDiagram
    participant D as Driver Process
    participant OS as Object Store
    participant W1 as Worker 1
    participant W2 as Worker 2
    D->>D: pickle.dumps(my_function)
    D->>OS: Store serialized function
    OS->>W1: Send bytes
    OS->>W2: Send bytes
    W1->>W1: pickle.loads(bytes) → function
    W2->>W2: pickle.loads(bytes) → function
    W1->>W1: function(data_block_1)
    W2->>W2: function(data_block_2)
`} fallback="Ray serializes functions to bytes in the driver, sends them through the object store, and deserializes on each worker" />

Ray uses **cloudpickle** (an enhanced version of Python's pickle) and **Apache Arrow** for tabular data. But not everything can be pickled.

## Common unpicklable objects

These objects cannot be serialized and will cause `TypeError: cannot pickle` errors:

| Object Type | Why It Fails | Example |
|---|---|---|
| Database connections | Contains OS-level socket handles | `psycopg2.connect(...)` |
| File handles | OS file descriptors are process-specific | `open("file.txt")` |
| Thread locks | Locks are tied to a specific process | `threading.Lock()` |
| CUDA tensors | GPU memory pointers are device-specific | `tensor.cuda()` |
| Lambda closures | Often capture unpicklable variables | `lambda x: conn.query(x)` |
| Generator objects | Internal iteration state cannot be captured | `(x for x in range(10))` |

## The fix: class-based transforms

This is the most reliable pattern for avoiding serialization issues. The class itself is picklable because Ray just sends the class reference. The `__init__` method runs on each worker, creating unpicklable resources locally.

### Pattern 1: database connections

<CodeBlock code={`# BROKEN: Connection created in driver, captured by lambda
import psycopg2
conn = psycopg2.connect("postgresql://localhost/mydb")
ds = ds.map(lambda row: {
    "result": conn.execute(f"SELECT * FROM lookup WHERE id={row['id']}")
})
# TypeError: cannot pickle 'psycopg2.extensions.connection' object
# FIXED: Connection created per-worker in __init__
class DatabaseLookup:
    def __init__(self):
        import psycopg2
        self.conn = psycopg2.connect("postgresql://localhost/mydb")
        self.cursor = self.conn.cursor()
    def __call__(self, row):
        self.cursor.execute("SELECT * FROM lookup WHERE id=%s", (row["id"],))
        return {"id": row["id"], "result": self.cursor.fetchone()}
ds = ds.map(DatabaseLookup, compute=ray.data.ActorPoolStrategy(size=4))`} language="python" filename="fix_database.py" />

### Pattern 2: ML model loading

<CodeBlock code={`# BROKEN: Model loaded in driver, captured by closure
import torch
model = torch.load("model.pt")
model.eval()
ds = ds.map_batches(lambda batch: {"pred": model(batch["input"])})
# TypeError: cannot pickle 'torch.Tensor' object  (model weights)
# FIXED: Model loaded per-worker
class ModelPredictor:
    def __init__(self):
        import torch
        self.model = torch.load("model.pt")
        self.model.eval()
        self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
        self.model.to(self.device)
    def __call__(self, batch):
        import torch
        with torch.no_grad():
            inputs = torch.tensor(batch["input"]).to(self.device)
            preds = self.model(inputs).cpu().numpy()
        return {"pred": preds}
ds = ds.map_batches(ModelPredictor, compute=ray.data.ActorPoolStrategy(size=2), num_gpus=1, batch_size=256)`} language="python" filename="fix_model.py" />

### Pattern 3: file handles and clients

<CodeBlock code={`# BROKEN: HTTP client created in driver
import httpx
client = httpx.Client(timeout=30)
ds = ds.map(lambda row: {"response": client.get(row["url"]).text})
# TypeError: cannot pickle '_thread.lock' object (from httpx connection pool)
# FIXED: Client created per-worker
class HttpFetcher:
    def __init__(self):
        import httpx
        self.client = httpx.Client(timeout=30)
    def __call__(self, row):
        response = self.client.get(row["url"])
        return {"url": row["url"], "response": response.text, "status": response.status_code}
ds = ds.map(HttpFetcher, compute=ray.data.ActorPoolStrategy(size=8))`} language="python" filename="fix_http_client.py" />

## ray.put() for large shared objects

When you have a large read-only object (like a lookup table or model weights) that all workers need, use `ray.put()` to store it once in the object store. Without it, the object gets serialized separately for every single task.

<CodeBlock code={`import ray
import numpy as np
# A large lookup table that every worker needs
lookup_table = np.load("embeddings_500mb.npy")  # 500 MB
# BAD: lookup_table is serialized for every single task
ds = ds.map(lambda row: {
    "embedding": lookup_table[row["id"]]
})
# GOOD: Store once in object store, workers read with zero-copy
lookup_ref = ray.put(lookup_table)
def enrich_with_embedding(row, lookup=lookup_ref):
    table = ray.get(lookup)  # zero-copy on same node
    return {"id": row["id"], "embedding": table[row["id"]]}
ds = ds.map(enrich_with_embedding)`} language="python" filename="ray_put_example.py" />

## Custom serializers

For objects with known serialization patterns, you can register custom serializers so Ray knows how to handle them:

<CodeBlock code={`import ray
import numpy as np
# Example: Register a custom serializer for a specific class
class GPUModel:
    def __init__(self, weights):
        self.weights = weights  # numpy array
        self._gpu_handle = None  # unpicklable GPU resource
    def to_serializable(self):
        return {"weights": self.weights}
    @classmethod
    def from_serializable(cls, data):
        return cls(data["weights"])
# Register the custom serializer
ray.util.register_custom_serializer(
    GPUModel,
    serializer=lambda obj: obj.to_serializable(),
    deserializer=lambda data: GPUModel.from_serializable(data),
)`} language="python" filename="custom_serializer.py" />

## ActorPoolStrategy for stateful transforms

When your transform has expensive initialization (loading a model, opening a connection), use a class-based transform with `concurrency`. This creates persistent actor workers that initialize once and handle many batches.

<CodeBlock code={`import ray
# The class __init__ runs ONCE per actor, not once per batch
class ExpensiveTransform:
    def __init__(self):
        # This runs once when the actor starts
        import torch
        print("Loading 5 GB model... (happens once per worker)")
        self.model = torch.load("large_model.pt")
        self.model.eval()
        self.tokenizer = load_tokenizer("tokenizer/")
        print("Model loaded!")
    def __call__(self, batch):
        # This runs for every batch — model is already loaded
        tokens = self.tokenizer(batch["text"])
        predictions = self.model(tokens)
        return {"text": batch["text"], "prediction": predictions}
ds = ray.data.read_parquet("data/")
ds = ds.map_batches(
    ExpensiveTransform,
    compute=ray.data.ActorPoolStrategy(size=4),
    batch_size=256,
    num_gpus=1,       # each actor gets 1 GPU
)`} language="python" filename="actor_pool.py" />

## Serialization decision tree

Use this flowchart when you hit a serialization error:

<MermaidDiagram chart={`flowchart TD
    A["TypeError: cannot pickle"] --> B{What is unpicklable?}
    B -->|"Connection / Handle\n(DB, HTTP, file)"| C[Use class-based transform\ncreate resource in __init__]
    B -->|"Large read-only data\n(lookup table, config)"| D["Use ray.put() to store\nin object store once"]
    B -->|"ML Model"| E{Need GPU?}
    E -->|Yes| F["Class-based transform\nwith num_gpus=1"]
    E -->|No| G["Class-based transform\nwith compute=ActorPoolStrategy(size=N)"]
    B -->|"Lambda closure"| H[Replace with named function\nor class-based transform]
