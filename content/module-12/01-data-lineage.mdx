## Why Observability Matters

When a pipeline processes millions of rows across dozens of workers, you need answers to: Where is the bottleneck? Which operator is slow? How much memory is each stage using? Ray Data provides built-in observability to answer these questions.

## Dataset Stats API

After executing a pipeline, Ray Data exposes detailed statistics for every operator. This is your first tool for debugging performance issues.

<CodeBlock language="python" code={`import ray

ds = ray.data.read_parquet("s3://my-bucket/events/")

ds = ds.map_batches(parse_events, batch_size=1024)
ds = ds.filter(lambda row: row["status"] == "active")
ds = ds.map_batches(enrich_features, batch_size=512)

# Materialize and inspect stats
ds = ds.materialize()
print(ds.stats())
# Dataset Stats:
#   Operator 1 ReadParquet: 200 blocks executed in 4.2s
#     * Output: 200 blocks, 50.0GB
#   Operator 2 MapBatches(parse_events): 200 blocks in 8.1s
#     * Throughput: 24.7 blocks/s
#   Operator 3 Filter: 200 blocks in 1.3s
#     * Rows filtered: 1,200,000 / 5,000,000
#   Operator 4 MapBatches(enrich_features): 200 blocks in 12.5s
#     * Throughput: 16.0 blocks/s`} />

## The Ray Dashboard

The Ray Dashboard provides a live, visual view of your pipeline. Navigate to the **Datasets** tab to see:

- **Operator progress** -- how many blocks each operator has processed
- **Throughput** -- rows/sec and bytes/sec per operator
- **Memory usage** -- object store memory consumed by each stage
- **Backpressure indicators** -- which operators are waiting on downstream capacity

## Debugging with Lineage

When an output looks wrong, lineage helps you trace back to the cause. Combine `stats()` with logging inside your UDFs to build a full picture.

<CodeBlock language="python" code={`import logging

logger = logging.getLogger("ray.data")
logger.setLevel(logging.DEBUG)

def parse_events(batch):
    # Log batch-level diagnostics for traceability
    logger.debug(
        f"parse_events: batch_size={len(batch['raw'])}, "
        f"null_count={batch['raw'].isna().sum()}"
    )
    batch["parsed"] = batch["raw"].apply(json.loads)
    return batch

ds = ray.data.read_parquet("s3://my-bucket/events/")
ds = ds.map_batches(parse_events, batch_size=1024)
ds.materialize()

# Check execution stats for anomalies
stats = ds.stats()
print(stats)  # Look for skewed block sizes or slow operators`} />

## Key Takeaways

- **`ds.stats()`** gives you per-operator metrics after materialization
- The **Ray Dashboard Datasets tab** shows live progress, throughput, and memory
- Add **structured logging** inside UDFs to trace data quality issues
- Lineage is about connecting outputs back to inputs -- essential for debugging production pipelines
