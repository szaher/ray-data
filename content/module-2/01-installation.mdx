## What you need

Before writing any Ray code, you need three things: **Python 3.9+**, a **virtual environment**, and the **ray[data] package**.

## Step 1: Check your Python version

Ray 2.55 requires **Python 3.9 or higher**. Python 3.10 and 3.11 are recommended for the best compatibility. Check your version:

<CodeBlock code={`python3 --version
# Python 3.11.9  ← You need 3.9+`} language="bash" filename="terminal" />

If your version is below 3.9, install a newer Python before continuing. On macOS, `brew install python@3.11` works well. On Ubuntu, use `sudo apt install python3.11`.

## Step 2: Create a virtual environment

Always isolate your Ray installation in a virtual environment. This prevents conflicts with other projects on your system.

**Option A: venv (built-in)**

<CodeBlock code={`# Create the environment
python3 -m venv ray-env
# Activate it
source ray-env/bin/activate    # macOS / Linux
ray-env\\Scripts\\activate       # Windows`} language="bash" filename="terminal" />

**Option B: conda**

<CodeBlock code={`# Create the environment with a specific Python version
conda create -n ray-env python=3.11 -y
# Activate it
conda activate ray-env`} language="bash" filename="terminal" />

Either approach works. Just make sure your environment is active before you install Ray.

## Step 3: Install Ray with data support

Ray is distributed as a single PyPI package with optional extras. The `[data]` extra pulls in everything you need for data processing:

<CodeBlock code={`# Install Ray with data processing support
pip install "ray[data]==2.55.1"
# This installs: ray, pyarrow, pandas, numpy, fsspec`} language="bash" filename="terminal" />

Pin the version with `==2.55.1` so your environment is reproducible. Without pinning, a future `pip install --upgrade` could pull a breaking change.

You can install multiple extras at once: `pip install "ray[data,train]==2.55.1"`. For this course, `ray[data]` is all you need.

## Step 4: Verify your installation

Run two quick checks to confirm everything works:

<CodeBlock code={`# Check 1: CLI version
ray --version
# ray, version 2.55.1
# Check 2: Python import
python3 -c "import ray; print(ray.__version__)"
# 2.55.1`} language="bash" filename="terminal" />

If both commands print `2.55.1`, you're ready. If you see an `ImportError`, make sure your virtual environment is activated.

## Platform-specific notes

**macOS (Apple Silicon M1/M2/M3):** Ray has native ARM wheels since version 2.4. Install normally with pip, no Rosetta or special flags needed. If you hit issues, make sure you're using a native ARM Python (not an x86 version under Rosetta).

**Linux:** Ray works out of the box on x86_64. For ARM64 Linux (e.g., Graviton instances), native wheels are available starting with Ray 2.7.

**Windows:** Ray does not run natively on Windows. Use **WSL2** (Windows Subsystem for Linux) instead. Install Ubuntu in WSL2, then follow the Linux instructions above.

<CodeBlock code={`# Windows users: install WSL2 first
wsl --install -d Ubuntu-22.04
# Then open the Ubuntu terminal and proceed with Linux instructions`} language="bash" filename="terminal" />

## What got installed?

When you ran `pip install "ray[data]"`, pip installed these key packages:

- **ray**: the core distributed runtime (tasks, actors, object store)
- **pyarrow**: columnar data format, the backbone of Ray Data
- **pandas**: DataFrame library used for batch processing
- **numpy**: numerical arrays
- **fsspec**: filesystem abstraction for reading from S3, GCS, local disk

You now have a complete environment for distributed data processing. Next up, you'll use it to run your first Ray program.
