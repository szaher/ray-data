## Your first Ray program in 5 lines

You have Ray installed. Now put it to use. The goal: run a function on a Ray worker instead of your local Python process.

<CodeBlock code={`import ray
ray.init()
@ray.remote
def hello(name):
    return f"Hello, {name}! I ran on a Ray worker."
result = ray.get(hello.remote("Alice"))
print(result)
# Hello, Alice! I ran on a Ray worker.`} language="python" filename="first_ray.py" />

Six lines and you've distributed a function. Here's what each piece does.

## What ray.init() does

`ray.init()` starts a **local Ray cluster** on your machine. Behind the scenes, it launches several processes:

- A **driver** process, your Python script
- **Worker** processes, one per CPU core, ready to execute tasks
- An **object store** for shared memory between workers
- The **Global Control Store (GCS)** for tracking cluster state
- A **dashboard** web UI at `http://127.0.0.1:8265`

## The @ray.remote decorator

The `@ray.remote` decorator tells Ray: "This function can run on any worker." It doesn't change what the function does. It changes **where** and **how** it runs.

<CodeBlock code={`# Without @ray.remote: runs locally on the driver
def square(x):
    return x * x
result = square(4)  # 16, runs right here
# With @ray.remote: runs on a Ray worker
@ray.remote
def square(x):
    return x * x
ref = square.remote(4)  # Returns immediately with an ObjectRef
result = ray.get(ref)   # Blocks until the worker finishes → 16`} language="python" filename="remote_decorator.py" />

Notice the key difference: `square.remote(4)` does **not** return `16`. It returns an **ObjectRef**, a future that points to the result in the object store. You need `ray.get()` to retrieve the actual value.

## ray.get() retrieves results

`ray.get()` blocks until the result is ready, then pulls it from the object store into your driver process. You can pass a single ref or a list:

<CodeBlock code={`import ray
ray.init()
@ray.remote
def double(x):
    return x * 2
# Submit 4 tasks in parallel
refs = [double.remote(i) for i in range(4)]
# Wait for all of them and collect results
results = ray.get(refs)
print(results)  # [0, 2, 4, 6]`} language="python" filename="parallel_tasks.py" />

All four calls to `double.remote()` return instantly. Ray schedules them across your CPU cores. `ray.get(refs)` waits for all four to finish and returns the results in order.

## Common first-time gotchas

These three mistakes trip up every beginner.

**Gotcha 1: Forgetting .remote()**

<CodeBlock code={`@ray.remote
def greet(name):
    return f"Hi {name}"
# WRONG: This calls the function locally, bypassing Ray entirely
result = greet("Alice")
# RIGHT: This dispatches the function to a Ray worker
ref = greet.remote("Alice")
result = ray.get(ref)`} language="python" filename="gotcha_remote.py" />

**Gotcha 2: Forgetting ray.get()**

<CodeBlock code={`@ray.remote
def compute():
    return 42
ref = compute.remote()
# WRONG: ref is an ObjectRef, not 42
print(ref)           # ObjectRef(...)
print(ref + 1)       # TypeError!
# RIGHT: use ray.get() to get the actual value
value = ray.get(ref)
print(value)         # 42
print(value + 1)     # 43`} language="python" filename="gotcha_get.py" />

**Gotcha 3: Calling ray.get() too early in a loop**

<CodeBlock code={`# SLOW: Submits one task, waits, submits next — no parallelism!
results = []
for i in range(100):
    ref = double.remote(i)
    results.append(ray.get(ref))  # Blocks every iteration
# FAST: Submit all tasks first, then wait once
refs = [double.remote(i) for i in range(100)]
results = ray.get(refs)  # All 100 run in parallel`} language="python" filename="gotcha_loop.py" />

The pattern to remember: **submit many, get once**. Fire off all your `.remote()` calls first, collect the refs, then call `ray.get()` on the whole list.

## Shutting down

When your script finishes, shut down Ray to free system resources:

<CodeBlock code={`ray.shutdown()
# Or use a context manager for automatic cleanup
with ray.init():
    ref = hello.remote("world")
    print(ray.get(ref))
# Ray is shut down here automatically`} language="python" filename="shutdown.py" />
