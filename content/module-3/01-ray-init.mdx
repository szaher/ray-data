## Starting Ray

Every Ray program begins with `ray.init()`. This call connects your Python process to a Ray cluster, or creates one if none exists.

<CodeBlock language="python" code={`import ray

# Start a local cluster (development)
ray.init()

# Connect to an existing cluster
ray.init(address="ray://my-cluster:10001")

# With runtime environment
ray.init(runtime_env={"pip": ["pandas", "pyarrow"]})
`} />

## Local mode vs cluster mode

When you call `ray.init()` with no address, Ray starts a **local cluster** on your machine. Every CPU core becomes a worker. Good for development and testing.

In production, you connect to a **remote cluster**, where multiple machines are coordinated by a head node.

## Anatomy of a Ray cluster

A Ray cluster has two types of nodes.

The **head node** runs the Global Control Store (GCS), the dashboard, and the autoscaler. There is exactly one head node per cluster.

**Worker nodes** run your tasks and actors. Each node has a **raylet** (scheduler + resource manager) and a local **object store** (shared memory).

## The Ray dashboard

Every cluster exposes a dashboard at **port 8265**. It shows running jobs, resource utilization, logs, and the state of actors and tasks.

<CodeBlock language="python" code={`import ray

context = ray.init()
print(context.dashboard_url)  # "127.0.0.1:8265"

# Check cluster resources
print(ray.cluster_resources())
# {'CPU': 8, 'memory': 17179869184, 'node:127.0.0.1': 1.0}
`} />

## Shutting down

Shut down Ray when you're done to free resources:

<CodeBlock language="python" code={`# Explicit shutdown
ray.shutdown()

# Or use a context manager
with ray.init() as context:
    # Ray is running here
    pass
# Ray is automatically shut down here
`} />

Ray also supports **runtime environments**. You can specify pip packages, environment variables, and working directories that get installed on every worker, ensuring reproducibility across the cluster.
