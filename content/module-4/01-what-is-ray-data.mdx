## Last-mile data processing

Ray Data is a distributed data processing library built on Ray, designed for **last-mile data preprocessing** in machine learning.

It sits between your raw data (files on S3, databases, etc.) and your ML training loop or inference pipeline.

## When to use Ray Data

| Scenario | Ray Data | Spark | Dask |
|----------|----------|-------|------|
| ML preprocessing (images, text) | Best fit | Overhead | Possible |
| Batch inference at scale | Best fit | Poor fit | Possible |
| ETL into a data warehouse | Not ideal | Best fit | Possible |
| Ad-hoc SQL analytics | Not ideal | Best fit | Not ideal |
| Streaming ML pipelines | Good fit | Good fit | Poor fit |

## Key features

- **Lazy execution** -- transformations are deferred until you consume results, enabling optimization.
- **Arrow-native** -- data is stored in Apache Arrow format for zero-copy reads and efficient memory use.
- **Heterogeneous compute** -- you can attach GPUs to specific transform steps while keeping others on CPU.
- **Streaming execution** -- data flows through a pipeline so the full dataset never needs to fit in memory.

## The core abstraction: Dataset

The central object in Ray Data is `Dataset`. A Dataset is a distributed collection of rows, split into **blocks** that are processed in parallel.

<CodeBlock language="python" code={`import ray

# Create a Dataset — this is lazy, no data is read yet
ds = ray.data.read_csv("s3://my-bucket/data/")

# Chain transformations — still lazy
ds = ds.map_batches(preprocess)
ds = ds.filter(lambda row: row["label"] is not None)

# Consume — this triggers execution
for batch in ds.iter_batches(batch_size=256):
    model.train(batch)`} />

You define **what** to do. Ray Data decides **how** to distribute, schedule, and pipeline the work.
