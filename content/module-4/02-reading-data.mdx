## Data sources

Ray Data reads from local files, cloud storage (S3, GCS, Azure Blob), and in-memory Python objects. Every reader returns a `Dataset`.

## Reading structured data

<CodeBlock language="python" code={`import ray

# CSV — single file or directory of files
ds = ray.data.read_csv("s3://bucket/data/users.csv")
ds = ray.data.read_csv("s3://bucket/data/csvs/")  # reads all CSVs in directory

# Parquet — columnar format, best performance
ds = ray.data.read_parquet("data/events.parquet")
ds = ray.data.read_parquet("s3://bucket/events/", columns=["user_id", "action"])

# JSON — line-delimited JSON files
ds = ray.data.read_json("logs/2024-01-*.json")`} />

## Reading images

For ML pipelines that process images, `read_images` loads files as numpy arrays.

<CodeBlock language="python" code={`# Read all images from a directory
ds = ray.data.read_images("s3://bucket/training-images/")

# Resize on read for consistency
ds = ray.data.read_images("s3://bucket/images/", size=(224, 224))

# Each row has an "image" column containing a numpy array
ds.schema()
# Column  Type
# ------  ----
# image   numpy.ndarray(shape=(224, 224, 3), dtype=uint8)`} />

## Creating datasets from Python

`from_items` works well for small in-memory data, useful for testing and prototyping.

<CodeBlock language="python" code={`# From a list of dictionaries
ds = ray.data.from_items([
    {"name": "Alice", "age": 30},
    {"name": "Bob", "age": 25},
    {"name": "Carol", "age": 35},
])

# Check what you have
ds.show(3)
# {'name': 'Alice', 'age': 30}
# {'name': 'Bob', 'age': 25}
# {'name': 'Carol', 'age': 35}`} />

## Choosing the right format

Parquet is the best choice for most production use cases. It is columnar, compressed, and supports predicate pushdown, so Ray Data reads only the columns and row groups you need.

CSV and JSON work but are slower and use more memory. Use them when your upstream system produces them and converting is not an option.

`read_images` is the right choice when your data is a directory of image files, which is common in computer vision workflows.
