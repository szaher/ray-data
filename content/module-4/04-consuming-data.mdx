## From Dataset to results

After building a transformation pipeline, you consume the results. Ray Data provides two categories of consumers: **iterators** for streaming and **writers** for persistence.

## Peeking at data: show and take

Use these for quick inspection during development. They trigger execution but only read a small amount of data.

<CodeBlock language="python" code={`import ray

ds = ray.data.read_csv("data/users.csv")

# show() prints the first N rows to stdout
ds.show(3)
# {'name': 'Alice', 'age': 30, 'city': 'NYC'}
# {'name': 'Bob', 'age': 25, 'city': 'LA'}
# {'name': 'Carol', 'age': 35, 'city': 'NYC'}

# take() returns a list of row dicts — useful in notebooks
rows = ds.take(5)
print(rows[0])  # {'name': 'Alice', 'age': 30, 'city': 'NYC'}`} />

## Streaming iteration: iter_rows and iter_batches

For production workloads, iterate through data in a streaming fashion. This avoids loading the entire dataset into memory.

<CodeBlock language="python" code={`# iter_rows — one row at a time (dict)
for row in ds.iter_rows():
    process(row["name"], row["age"])

# iter_batches — batch of rows as DataFrame or numpy arrays
for batch in ds.iter_batches(batch_size=256, batch_format="pandas"):
    print(batch.shape)  # (256, 3)

# For ML training — numpy format for easy tensor conversion
for batch in ds.iter_batches(batch_size=64, batch_format="numpy"):
    features = batch["features"]  # numpy array
    labels = batch["labels"]      # numpy array
    # Convert to torch tensors, train your model, etc.`} />

## Writing data to disk

Write processed data out to files. Ray Data writes in parallel, creating one file per block.

<CodeBlock language="python" code={`# Write to Parquet — best for downstream ML pipelines
ds.write_parquet("s3://bucket/processed/output/")

# Write to CSV
ds.write_csv("/tmp/output_csvs/")

# Write to JSON
ds.write_json("/tmp/output_json/")

# Control the number of output files
ds.repartition(10).write_parquet("s3://bucket/output/")`} />

## The ML training pattern

The most common consumption pattern is feeding an ML training loop:

<CodeBlock language="python" code={`import ray

ds = ray.data.read_parquet("s3://bucket/training-data/")
ds = ds.map_batches(preprocess, batch_format="pandas")

# Stream data epoch by epoch
for epoch in range(10):
    for batch in ds.iter_batches(batch_size=128, batch_format="numpy"):
        features = batch["features"]
        labels = batch["labels"]
        loss = train_step(features, labels)
    print(f"Epoch {epoch}: loss={loss:.4f}")`} />

## Consumer summary

| Method | Returns | Use Case |
|--------|---------|----------|
| `show(n)` | None (prints) | Quick inspection |
| `take(n)` | List of dicts | Grab rows for debugging |
| `iter_rows()` | Iterator of dicts | Row-level streaming |
| `iter_batches()` | Iterator of batches | ML training, batch processing |
| `write_parquet()` | None (writes files) | Persist for downstream use |
| `write_csv()` | None (writes files) | Export to CSV format |
