## Arrow under the hood

Ray Data stores all data internally as **Apache Arrow tables**. When you read a CSV, Parquet file, or create a Dataset from Python objects, the data is converted to Arrow format.

Arrow is columnar, supports zero-copy reads between processes, and provides a strict type system.

## Inspecting the schema

Every Dataset has a schema describing its columns and types. Use `.schema()` to inspect it.

<CodeBlock language="python" code={`import ray

ds = ray.data.from_items([
    {"name": "Alice", "age": 30, "score": 95.5},
    {"name": "Bob", "age": 25, "score": 87.0},
])

print(ds.schema())
# Column  Type
# ------  ----
# name    string
# age     int64
# score   double`} />

## Arrow type mapping

| Python Type | Arrow Type | Notes |
|-------------|------------|-------|
| `str` | `string` | UTF-8 encoded |
| `int` | `int64` | 64-bit integer |
| `float` | `double` | 64-bit float |
| `bool` | `bool` | True/False |
| `bytes` | `binary` | Raw bytes |
| `list` | `list<T>` | Typed lists |
| `np.ndarray` | `ArrowTensorType` | For ML tensors |

## Batch formats in practice

When calling `map_batches`, the `batch_format` parameter controls what your function receives and should return.

<CodeBlock language="python" code={`import pandas as pd
import pyarrow as pa

ds = ray.data.read_parquet("data/events.parquet")

# pandas — familiar API, good for most transforms
def transform_pandas(batch: pd.DataFrame) -> pd.DataFrame:
    batch["event_upper"] = batch["event"].str.upper()
    return batch

ds.map_batches(transform_pandas, batch_format="pandas")

# pyarrow — fastest, best for column operations and zero-copy
def transform_arrow(table: pa.Table) -> pa.Table:
    col = pa.compute.utf8_upper(table.column("event"))
    return table.append_column("event_upper", col)

ds.map_batches(transform_arrow, batch_format="pyarrow")

# numpy — dict of arrays, ideal for feeding ML models
def transform_numpy(batch: dict) -> dict:
    batch["features_scaled"] = batch["features"] / 255.0
    return batch

ds.map_batches(transform_numpy, batch_format="numpy")`} />

## When schema matters

Schema mismatches cause runtime errors. Common pitfalls:

1. Returning different columns from `map_batches` than what came in. This is fine, but be consistent across batches.
2. Mixing types. If one batch returns `int` and another returns `float` for the same column, Arrow will reject it.
3. Null handling. Arrow distinguishes between `None` and missing values. Use `pa.array([1, None, 3])` to include nulls explicitly.

<CodeBlock language="python" code={`# Check schema before and after a transform
ds = ray.data.read_csv("data.csv")
print("Before:", ds.schema())

ds = ds.map_batches(my_transform, batch_format="pandas")
print("After:", ds.schema())

# count() triggers execution and returns total rows
print("Rows:", ds.count())`} />

## Choosing a batch format

- **pandas** -- Good default. Wide library support, great for string and datetime operations.
- **pyarrow** -- Fastest for pure column transforms. Use when performance matters and you are comfortable with the Arrow compute API.
- **numpy** -- Natural fit for numeric ML features. Returns a `dict[str, np.ndarray]`.
