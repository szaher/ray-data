## Bulk vs. streaming

Ray Data uses streaming execution by default. It pipelines operators so that reading, transforming, and writing overlap in time. Compare this to bulk execution, where each step must finish completely before the next one starts.

## How streaming works

Ray Data divides data into blocks and pipelines them through operators. While block N is being transformed, block N+1 is being read, and block N-1 is being written.

<CodeBlock language="python" code={`import ray

# Streaming is the default -- this pipeline never materializes all data
ds = ray.data.read_parquet("s3://bucket/terabyte-dataset/")

ds = ds.map_batches(preprocess, batch_size=4096)
ds = ds.map_batches(run_model, batch_size=512, num_gpus=1)

# Data flows through read -> preprocess -> model -> write
ds.write_parquet("s3://bucket/output/")`} />

## Backpressure and flow control

When operators run at different speeds, Ray Data applies backpressure, slowing fast producers to match slow consumers. This prevents memory from growing without bound.

## Controlling execution

You can tune streaming behavior with `ExecutionOptions`.

<CodeBlock language="python" code={`import ray
from ray.data import ExecutionOptions

ctx = ray.data.DataContext.get_current()

# Limit memory to avoid OOM on large pipelines
ctx.execution_options = ExecutionOptions(
    resource_limits=ray.data.ExecutionResources(
        object_store_memory=4 * 1024 * 1024 * 1024,  # 4 GB cap
    ),
)

# Pipeline will stay within the 4 GB memory budget
ds = ray.data.read_parquet("s3://bucket/huge-dataset/")
ds.map_batches(transform).write_parquet("s3://bucket/out/")`} />

## When to force materialization

Sometimes you need the full dataset in memory. Global shuffling and multi-pass iteration (training epochs, for example) both require it. Use `materialize()` explicitly for these cases.

<CodeBlock language="python" code={`# Force full materialization when needed
ds = ray.data.read_parquet("s3://bucket/data/")
ds = ds.map_batches(transform)

# Materialize into object store (dataset must fit in cluster memory)
ds = ds.materialize()

# Now you can iterate multiple times without re-reading
for epoch in range(10):
    for batch in ds.iter_batches(batch_size=256):
        train_step(batch)`} />

Prefer streaming unless you have a specific reason to materialize. It is the default for good reason.
