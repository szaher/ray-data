## The data loading bottleneck

In distributed training, each GPU worker needs a continuous stream of preprocessed batches. If data loading is slow, GPUs sit idle. Ray Data addresses this by running preprocessing on separate CPU workers and streaming results to trainers.

## Passing datasets to a trainer

Create your Ray Dataset outside the trainer, then pass it via the `datasets` argument. Ray Train handles sharding automatically.

<CodeBlock language="python" code={`import ray
from ray.train.torch import TorchTrainer
from ray.train import ScalingConfig

# 1. Create and preprocess the dataset
train_ds = ray.data.read_parquet("s3://bucket/train/")
train_ds = train_ds.map_batches(normalize_features, batch_size=4096)

val_ds = ray.data.read_parquet("s3://bucket/val/")

# 2. Pass datasets to the trainer
trainer = TorchTrainer(
    train_loop_per_worker=train_fn,
    datasets={"train": train_ds, "val": val_ds},
    # gpu-example: train-scaling
    scaling_config=ScalingConfig(
        num_workers=4,
        use_gpu=True,
    ),
)
result = trainer.fit()`} />

For CPU-only experimentation:

<CodeBlock language="python" code={`# cpu-alternative: train-scaling

## The training function

Inside each worker, `get_dataset_shard` returns that worker's portion of the data. Use `iter_torch_batches` to get PyTorch tensors directly.

<CodeBlock language="python" code={`import ray.train
from ray.train import report

def train_fn(config):
    import torch
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = build_model().to(device)
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

    # Each worker gets its own shard automatically
    train_shard = ray.train.get_dataset_shard("train")

    for epoch in range(config.get("epochs", 10)):
        for batch in train_shard.iter_torch_batches(
            batch_size=256,
            dtypes=torch.float32,
        ):
            features = batch["features"].to(device)
            labels = batch["labels"].to(device)

            loss = train_step(model, optimizer, features, labels)

        report({"loss": loss.item(), "epoch": epoch})`} />
