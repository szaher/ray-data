## From Training to Serving

A common production pitfall is **training-serving skew** -- the model was trained with one preprocessing pipeline, but serves with a different one. Ray Data lets you define preprocessing once and reuse it in both contexts.

## A Basic Ray Serve Deployment

Ray Serve deployments are Python classes decorated with `@serve.deployment`. You load the model in `__init__` and handle requests in `__call__`.

<CodeBlock language="python" code={`import ray
from ray import serve
import numpy as np

@serve.deployment(num_replicas=2, ray_actor_options={"num_gpus": 1})
class TextClassifier:
    def __init__(self):
        from transformers import pipeline
        self.model = pipeline(
            "text-classification",
            model="distilbert-base-uncased-finetuned-sst-2-english",
            device=0,
        )

    async def __call__(self, request):
        data = await request.json()
        text = data["text"]
        result = self.model(text)
        return {"label": result[0]["label"], "score": result[0]["score"]}

app = TextClassifier.bind()
serve.run(app)`} />

## Composing a Multi-Stage Pipeline

For multi-step pipelines, compose deployments using `.bind()`. The preprocessor handles tokenization before the model runs inference.

<CodeBlock language="python" code={`@serve.deployment(num_replicas=2)
class Preprocessor:
    def __init__(self):
        from transformers import AutoTokenizer
        self.tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")

    async def __call__(self, text: str) -> dict:
        return self.tokenizer(text, padding="max_length", truncation=True, max_length=128)

@serve.deployment(num_replicas=2, ray_actor_options={"num_gpus": 1})
class ModelInference:
    def __init__(self, preprocessor):
        import torch
        from transformers import AutoModelForSequenceClassification
        self.model = AutoModelForSequenceClassification.from_pretrained(
            "distilbert-base-uncased-finetuned-sst-2-english"
        ).cuda().eval()
        self.preprocessor = preprocessor

    async def __call__(self, request):
        import torch
        data = await request.json()
        tokens = await self.preprocessor.remote(data["text"])
        input_ids = torch.tensor([tokens["input_ids"]]).cuda()
        with torch.no_grad():
            pred = self.model(input_ids).logits.argmax(dim=1).item()
        return {"class": pred}

preprocessor = Preprocessor.bind()
app = ModelInference.bind(preprocessor)
serve.run(app)`} />

**Key pattern:** CPU-intensive preprocessing scales on cheap CPU replicas, while expensive GPU inference runs on dedicated GPU replicas. Ray Serve autoscales each stage independently based on load.
