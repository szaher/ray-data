## Why preprocessors?

ML pipelines need consistent data transformations at training and inference time. Ray Data preprocessors encapsulate transformations as portable, composable objects. You fit them on training data once and apply them identically during serving.

## Built-in preprocessors

Ray Data ships with common ML preprocessors. They follow a fit/transform pattern: learn statistics from training data, then apply them consistently.

<CodeBlock language="python" code={`from ray.data.preprocessors import StandardScaler, MinMaxScaler, OrdinalEncoder

# StandardScaler: zero mean, unit variance
scaler = StandardScaler(columns=["age", "income", "score"])

# MinMaxScaler: scale to [0, 1] range
minmax = MinMaxScaler(columns=["price", "quantity"])

# OrdinalEncoder: convert categories to integers
encoder = OrdinalEncoder(columns=["city", "product_type"])

# Fit on training data, then transform
ds = ray.data.read_parquet("s3://bucket/train/")
scaler.fit(ds)
transformed = scaler.transform(ds)`} />

## Custom transforms with BatchMapper

When built-in preprocessors don't cover your use case, `BatchMapper` lets you write arbitrary transformation functions that operate on batches.

<CodeBlock language="python" code={`from ray.data.preprocessors import BatchMapper
import numpy as np

# Custom feature engineering
def add_features(batch: dict) -> dict:
    batch["price_per_sqft"] = batch["price"] / batch["sqft"]
    batch["log_income"] = np.log1p(batch["income"])
    batch["age_bucket"] = (batch["age"] // 10) * 10  # decade buckets
    return batch

feature_eng = BatchMapper(add_features, batch_format="numpy")`} />

## Composing with Chain

`Chain` combines multiple preprocessors into a single pipeline. Order matters: each preprocessor's output feeds into the next.

<CodeBlock language="python" code={`from ray.data.preprocessors import Chain, StandardScaler, OrdinalEncoder

# Build a reusable preprocessing pipeline
preprocessor = Chain(
    OrdinalEncoder(columns=["city", "category"]),
    BatchMapper(add_features, batch_format="numpy"),
    StandardScaler(columns=["price_per_sqft", "log_income", "score"]),
)

# Fit the entire chain on training data
train_ds = ray.data.read_parquet("s3://bucket/train/")
preprocessor.fit(train_ds)
train_processed = preprocessor.transform(train_ds)

# Apply the same chain at inference time -- guarantees consistency
test_ds = ray.data.read_parquet("s3://bucket/test/")
test_processed = preprocessor.transform(test_ds)`} />

## Preprocessors with Ray Train

Pass the preprocessor directly to the Trainer. It gets applied automatically and saved with checkpoints.

<CodeBlock language="python" code={`from ray.train.torch import TorchTrainer
from ray.train import ScalingConfig

trainer = TorchTrainer(
    train_loop_per_worker=train_fn,
    datasets={"train": train_ds, "val": val_ds},
    preprocessor=preprocessor,  # applied before data reaches workers
    # gpu-example: preprocessor-train
    scaling_config=ScalingConfig(num_workers=4, use_gpu=True),
)
result = trainer.fit()`} />

For CPU-only experimentation:

<CodeBlock language="python" code={`# cpu-alternative: preprocessor-train
scaling_config=ScalingConfig(num_workers=2, use_gpu=False)`} />

Define your preprocessing pipeline once with `Chain`, fit it on training data, and reuse the same fitted pipeline everywhere: training, evaluation, and serving.
