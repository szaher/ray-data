## Scaling LLM Inference with Ray Data

Running an LLM over millions of prompts is a perfect Ray Data workload: each prompt is independent, GPU utilization is critical, and batching improves throughput.

## Approach 1 — HuggingFace Transformers

Ideal for models that fit on a single GPU.

<CodeBlock language="python" code={`import ray

ds = ray.data.read_parquet("s3://data/prompts/")

class HFPredictor:
    def __init__(self):
        from transformers import pipeline
        self.pipe = pipeline(
            "text-generation", model="meta-llama/Llama-3.1-8B-Instruct",
            device="cuda", torch_dtype="auto",
        )

    def __call__(self, batch: dict) -> dict:
        outputs = self.pipe(batch["prompt"].tolist(), max_new_tokens=256,
                            batch_size=len(batch["prompt"]))
        batch["response"] = [o[0]["generated_text"] for o in outputs]
        return batch

ds = ds.map_batches(HFPredictor, batch_size=32, concurrency=4, num_gpus=1)
ds.write_parquet("s3://data/results/")
`} />

## Approach 2 — vLLM for High Throughput

vLLM uses PagedAttention and continuous batching. It supports tensor parallelism across multiple GPUs for large models.

<CodeBlock language="python" code={`class VLLMPredictor:
    def __init__(self):
        from vllm import LLM, SamplingParams
        self.llm = LLM(model="meta-llama/Llama-3.1-70B-Instruct",
                        tensor_parallel_size=4, max_model_len=4096)
        self.params = SamplingParams(temperature=0.7, max_tokens=512)

    def __call__(self, batch: dict) -> dict:
        outputs = self.llm.generate(batch["prompt"].tolist(), self.params)
        batch["response"] = [o.outputs[0].text for o in outputs]
        return batch

ds.map_batches(VLLMPredictor, batch_size=256, concurrency=2, num_gpus=4)
`} />

**Batch size guidance:** HuggingFace works best with 8-64 (memory-bound). vLLM handles 128-512 efficiently via internal continuous batching. Increase until GPU memory hits ~90%.
