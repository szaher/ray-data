## Building a Production ETL Pipeline

ETL is the bread and butter of data engineering. Ray Data excels here because it streams data without materializing entire datasets in memory.

## Step 1 — Extract from Multiple Sources

<CodeBlock language="python" code={`import ray

events = ray.data.read_parquet("s3://datalake/raw/events/2025-06/")

# Materialize small lookup table for broadcast join
users_list = ray.data.read_csv("s3://datalake/dim/users.csv").take_all()
user_lookup = {u["user_id"]: u for u in users_list}
`} />

## Step 2 — Clean, Validate, Enrich

<CodeBlock language="python" code={`import numpy as np

def clean(batch: dict[str, np.ndarray]) -> dict[str, np.ndarray]:
    mask = batch["user_id"] != b""
    batch = {k: v[mask] for k, v in batch.items()}
    batch["event_type"] = np.array([e.decode().lower() for e in batch["event_type"]])
    return batch

def enrich(batch: dict) -> dict:
    countries, segments = [], []
    for uid in batch["user_id"]:
        user = user_lookup.get(uid.decode(), {})
        countries.append(user.get("country", "unknown"))
        segments.append(user.get("segment", "unknown"))
    batch["country"] = countries
    batch["segment"] = segments
    return batch

events = events.map_batches(clean, batch_format="numpy")
events = events.map_batches(enrich, batch_format="numpy")
`} />

## Step 3 — Write Partitioned Output

<CodeBlock language="python" code={`def add_date_partition(batch: dict) -> dict:
    batch["date"] = np.array([ts.decode()[:10] for ts in batch["timestamp"]])
    return batch

events = events.map_batches(add_date_partition, batch_format="numpy")
events.write_parquet("s3://datalake/curated/events/", partition_cols=["date", "country"])
`} />

Key takeaways: use the **broadcast-join pattern** (materialize small table, reference in `map_batches`) for enrichment; use **partition_cols** on write to enable downstream partition pruning.
