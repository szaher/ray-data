## From Batch Pipeline to Online Endpoint

A common challenge: you build preprocessing for training, then rewrite it for serving. With shared transforms, both paths stay in sync.

## Shared Preprocessing Logic

<CodeBlock language="python" code={`import numpy as np

def compute_features(batch: dict) -> dict:
    amounts = np.array(batch["amount"], dtype=np.float32)
    batch["log_amount"] = np.log1p(amounts)
    batch["is_high_value"] = (amounts > 1000).astype(np.int32)
    hours = np.array([int(t[11:13]) for t in batch["timestamp"]])
    batch["hour_sin"] = np.sin(2 * np.pi * hours / 24).astype(np.float32)
    batch["hour_cos"] = np.cos(2 * np.pi * hours / 24).astype(np.float32)
    return batch
`} />

## Ray Serve Deployment

<CodeBlock language="python" code={`from ray import serve
from starlette.requests import Request

@serve.deployment(num_replicas=2, ray_actor_options={"num_gpus": 1})
class FraudDetector:
    def __init__(self):
        import torch
        self.model = torch.jit.load("model.pt").eval().cuda()

    async def __call__(self, request: Request) -> dict:
        import torch
        payload = await request.json()
        features = compute_features({
            "amount": [payload["amount"]],
            "timestamp": [payload["timestamp"]],
        })
        tensor = torch.tensor([[
            features["log_amount"][0], features["is_high_value"][0],
            features["hour_sin"][0], features["hour_cos"][0],
        ]], dtype=torch.float32).cuda()
        with torch.no_grad():
            score = self.model(tensor).item()
        return {"fraud_score": score, "is_fraud": score > 0.5}

app = FraudDetector.bind()
serve.run(app, route_prefix="/predict")
`} />

The key insight: **compute_features** is identical in both batch and online paths. Update feature logic once and both pipelines stay in sync — eliminating training-serving skew.
